# AI 教学改革实训平台 — 需求文档

## 1. 项目背景

### 1.1 问题描述

在 AI 时代的大学编程课堂中，学生普遍习惯将老师布置的编程任务直接通过网页版 AI 工具获取完整答案。这种模式存在严重问题：

- 学生被动接受答案，无法真正理解和掌握知识
- 缺乏思考过程，无法培养问题分析和解决能力
- 传统答题模式无法提供个性化的学习引导
- 教师无法了解学生的真实学习情况和薄弱环节

### 1.2 解决方案

构建一个 AI 驱动的教学实训平台，核心改造思路：

**从"AI 直接给答案"转变为"AI 引导学生思考"**

- AI 助教基于知识库提供引导式帮助，而非直接给出完整答案
- 通过 Rubric 引擎进行规则+AI 双轨评分，确保评价客观性
- 教师端提供完整的学情分析、知识库管理、题目发布和反馈能力
- 双主题视觉设计：学生端 Liquid Glass 暖奶油风格，教师端马卡龙 HF 风格

---

## 2. 用户角色

### 2.1 学生

- 身份：高职院校软件技术专业学生
- 场景：课堂实训、课后练习
- 需求：理解任务要求、获得引导式帮助、查看学习记录、配置 AI 连接

### 2.2 教师

- 身份：授课教师、实训指导教师
- 需求：管理知识库、管理题库、发布题目、查看学生学习情况、分析薄弱环节、发送反馈、配置 AI 连接

## 6. 约束条件

### 6.1 技术约束

- Python 运行时：系统默认 Python 或 Anaconda
- 包管理器：npm（不使用 pnpm/yarn）
- 不引入重型基础设施：不使用 Milvus、Neo4j、Redis、RabbitMQ、Docker

### 6.2 业务约束

- MVP 阶段：无真实登录认证，身份切换仅用于演示
- 无真实 Java 代码沙箱
- DeepSeek 不可用时返回真实错误，不返回伪造答案
