# 05-迭代实施计划

## 一、项目1：需求与技术方案

交付内容：

- 项目需求分析
- 技术方案规划
- 数据库规划
- 接口规范
- 后续迭代计划

验收标准：

- 明确系统功能范围。
- 明确五个项目之间的迭代关系。
- 明确后端、前端、数据库技术路线。

## 二、项目2：SSM 后端基础功能

交付内容：

- SSM 后端源码。
- 系统基础功能。
- 基础业务 CRUD。

验收标准：

- 后端可编译。
- Controller、Service、Mapper 分层清晰。
- 具备基础接口能力。

## 三、项目3：Spring Boot 知识管理模块

交付内容：

- Spring Boot 后端。
- Vue 前端。
- 登录功能。
- 专家、图文课程、视频课程管理。

验收标准：

- 后端可启动。
- 前端可启动。
- 可使用 `admin / 123456` 登录。
- 登录后能进入知识管理页面。

## 四、项目4：Spring Boot 进阶业务模块

交付内容：

- 在项目3基础上增加补贴政策、信贷信息、农事服务。
- 前端菜单和路由同步增加对应页面。

验收标准：

- 后端可启动。
- 前端可启动。
- 可登录进入后台。
- 能访问知识管理和三个新增业务模块。

## 五、项目5：完整前后端对接

交付内容：

- 完整 Spring Boot 后端。
- 完整 Vue 前端。
- 登录、系统管理、知识管理、业务管理、农产品、市场、广告、数据统计完整对接。

验收标准：

- 前后端能完整联调。
- 页面能调用真实接口。
- 列表、新增、修改、删除、上传功能可用。
