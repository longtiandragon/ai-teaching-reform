# 项目1：解析项目需求与规划技术方案

本阶段用于明确农博后台管理系统的建设目标、功能范围、技术架构、数据库规划和后续迭代路线。

## 交付文件

```text
项目1-需求与技术方案
├── README.md
├── 01-项目需求分析.md
├── 02-技术方案规划.md
├── 03-数据库规划.md
├── 04-接口规范.md
└── 05-迭代实施计划.md
```

## 项目目标

建设一个面向农业服务场景的后台管理系统，支持管理员对农业知识、专家、图文课程、视频课程、补贴政策、信贷信息、农事服务、农产品、农贸市场、广告和系统基础数据进行统一维护。

## 五个阶段关系

| 阶段 | 内容 | 说明 |
| --- | --- | --- |
| 项目1 | 需求分析与技术方案 | 明确系统做什么、怎么做、怎么迭代 |
| 项目2 | SSM 后端基础功能 | 使用 SSM 实现基础后端 |
| 项目3 | Spring Boot 知识管理模块 | 可登录，保留专家、图文课程、视频课程 |
| 项目4 | Spring Boot 进阶业务模块 | 在项目3基础上增加补贴、信贷、农事服务 |
| 项目5 | 完整前后端对接 | 后端完整功能 + Vue 前端完整联调 |

## 推荐阅读顺序

1. 先看 `01-项目需求分析.md`，明确系统功能范围。
2. 再看 `02-技术方案规划.md`，明确后端、前端、数据库技术选型。
3. 再看 `03-数据库规划.md`，明确主要业务表。
4. 再看 `04-接口规范.md`，明确接口命名和返回格式。
5. 最后看 `05-迭代实施计划.md`，明确项目2到项目5如何逐步实现。
