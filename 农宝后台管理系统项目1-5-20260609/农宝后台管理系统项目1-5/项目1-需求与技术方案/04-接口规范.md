# 04-接口规范

## 一、接口风格

接口采用 REST 风格，统一使用 JSON 传输数据。

## 二、统一响应格式

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {},
  "total": 0
}
```

## 三、通用 CRUD 接口

| 操作 | 方法 | 地址示例 |
| --- | --- | --- |
| 查询列表 | GET | `/dev-api/yjnb/expert/list` |
| 查询详情 | GET | `/dev-api/yjnb/expert/{id}` |
| 新增 | POST | `/dev-api/yjnb/expert` |
| 修改 | PUT | `/dev-api/yjnb/expert` |
| 删除 | DELETE | `/dev-api/yjnb/expert/{ids}` |

## 四、登录接口

| 功能 | 方法 | 地址 |
| --- | --- | --- |
| 登录 | POST | `/dev-api/yjnb/system/login` |
| 退出 | POST | `/dev-api/yjnb/system/logout` |
| 用户信息 | GET | `/dev-api/yjnb/system/userInfo` |

登录请求示例：

```json
{
  "username": "admin",
  "password": "123456"
}
```
