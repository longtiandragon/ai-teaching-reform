# AI 教学改革实训平台 — 技术文档

## 1. 项目概述

本平台是一个面向高职院校《Web 系统应用开发》课程的 AI 驱动教学实训系统。核心目标是通过 AI 助教引导学生逐步完成学习任务，结合 Rubric 规则引擎进行过程性评价，同时为教师提供完整的学情分析、知识库管理和反馈能力。

### 1.1 核心能力

| 能力 | 说明 |
|------|------|
| 可配置课程线 | 支持多课程、多项目并行，课程内容数据驱动 |
| AI 实训助教 | 按任务目标和评分标准引导学生，而非直接给答案 |
| RAG 证据增强 | 回答和检查基于课程标准、项目需求、职业技能标准 |
| 过程性评价 | Rubric 引擎 60% 规则 + 40% AI 双轨评分 |
| 发布题目系统 | 教师将题库中的题目发布到指定任务关卡，学生在路线中答题 |
| 教师 AI 分析 | AI 自动分析学生/班级的学习数据，生成优势、不足和改进建议 |
| 反馈系统 | 教师向学生发送文字反馈，支持附带 AI 分析结果 |
| 知识库文件管理 | 上传 PDF/Word/Excel/CSV/代码文件，自动解析索引，关联任务 |
| 功能引导弹窗 | 首次进入平台时展示功能说明，帮助用户快速上手 |
| 双端视觉 | 学生端 Liquid Glass 暖奶油风格，教师端马卡龙 HF 风格 |

### 3.2 RAG 检索架构

```
学生提问
    │
    ▼
┌──────────────────────┐
│  关键词检索 (始终启用)  │ ← 分词 → BM25 打分 → 课程/概念加权
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  向量检索 (可选)       │ ← ChromaDB + fastembed (bge-small-en-v1.5)
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  合并去重 + 重排序     │ ← 按分数降序，取 top_k 条
│  返回 Citation 列表    │
└──────────────────────┘
```

**知识库数据来源 (290 个 chunks)：**

| 类型 | 数量 | 来源 |
|------|------|------|
| course-material | 134 | docs/course-materials/ 下的 12 份 Markdown 课件 |
| project-document | 66 | 农宝系统源码中的 README、Controller、Service 等 |
| lesson | 24 | 每课的教学目标和内容摘要 |
| practice | 24 | 每课的练习任务和检查清单 |
| database-schema | 26 | nb_database.sql 的表结构说明 |
| course-standard | 14 | 课程标准文档的关键段落 |
| standard | 1 | 职业技能等级标准 |

### 3.3 Rubric 评分引擎

```
final_score = rule_score × 0.6 + ai_score × 0.4
passed = final_score >= minScore AND NOT critical_failed
```

支持 6 种评判标准：keyword、regex、concept、rag_grounded、code_pattern、anti_shortcut。

### 3.4 文件上传解析架构

```
教师上传文件
    │
    ▼
┌──────────────────────┐
│  文件类型检测          │ ← 根据后缀名分发
└──────────┬───────────┘
           │
    ┌──────┼──────┬──────┬──────┐
    ▼      ▼      ▼      ▼      ▼
  .pdf   .docx  .xlsx   .csv   代码/文本
  pypdf  python  open-  csv    UTF-8
         docx   pyxl   模块   直接读取
    │      │      │      │      │
    └──────┴──────┴──────┴──────┘
           │
           ▼
┌──────────────────────┐
│  RAG 索引 + 元数据注册 │ ← rag_service.add_document()
│  kb_files 表记录       │   kb_manager.register_file()
└──────────────────────┘
```

支持的文件格式：
- **文档**: PDF (.pdf), Word (.docx), Excel (.xlsx, .xls), CSV (.csv)
- **代码**: .java, .py, .js, .ts, .vue, .jsx, .tsx, .xml, .sql, .html, .css 等
- **配置/文本**: .md, .txt, .yml, .yaml, .json, .properties, .toml, .ini 等

### 6.2 路由守卫

```typescript
router.beforeEach((to) => {
  if (to.path === '/login') return true
  if (!session.currentUser) return '/login'
  const requiredRole = to.meta.role
  if (requiredRole && requiredRole !== session.currentUser.role) {
    return session.currentUser.role === 'teacher' ? '/teacher' : '/student'
  }
  return true
})
```

**BorderLightFlow** — 边框光流导航特效
- SVG stroke-dasharray 动画
- 两条光流从顶部中心出发，沿边框反方向绕一圈
- 点击导航链接时触发，1.2 秒完成

- Node.js 18+
- npm

### 7.2 安装与运行

```powershell

python -m venv .venv
.venv\Scripts\pip install -r backend\requirements.txt
.venv\Scripts\python -m uvicorn backend.app.main:app --reload --host 127.0.0.1 --port 8001

```
DEEPSEEK_API_KEY=your_key_here
DEEPSEEK_BASE_URL=https://api.deepseek.com

DEEPSEEK_LIVE=true
