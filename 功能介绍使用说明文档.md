# AI 教学改革实训平台 — 功能介绍与使用说明

## 1. 平台简介

AI 教学改革实训平台是一个面向高职院校《Web 系统应用开发》课程的智能教学系统。平台通过 AI 助教帮助学生理解编程任务、逐步完成学习目标，同时为教师提供学情分析、知识库管理、题目发布和个性化反馈等管理能力。

**核心理念：** 从"AI 直接给答案"转变为"AI 引导学生思考"。

**适用对象：**
- 学生：高职院校软件技术专业学生，用于课堂实训和课后练习
- 教师：授课教师、实训指导教师，用于教学管理和学情分析

**平台特色：**
- AI 助教基于课程知识库回答问题，附带引用来源
- Rubric 规则引擎 + AI 双轨评分，确保评价客观性
- 学生端 Liquid Glass 暖奶油风格，教师端马卡龙 HF 风格
- 支持两条课程轨道并行：SpringBoot 12 讲 + 农宝系统后台管理项目

## 3. 安装与启动

### 3.1 克隆项目

```bash
git clone <仓库地址>
cd ai-teaching-reform
```

### 3.2 后端启动

```powershell
# 1. 创建 Python 虚拟环境
python -m venv .venv

# 2. 激活虚拟环境 (Windows)
.venv\Scripts\activate

# 3. 安装依赖
pip install -r backend/requirements.txt

# 4. (可选) 安装 ChromaDB 向量检索支持
pip install -r backend/requirements-optional-rag.txt

# 5. 配置环境变量 (见 3.4 节)

# 6. 启动后端服务 (默认端口 8001)
python -m uvicorn backend.app.main:app --reload --host 127.0.0.1 --port 8001
```

### 3.3 前端启动

```powershell
# 1. 进入前端目录
cd frontend

# 2. 安装依赖
npm install

# 3. 启动开发服务器 (默认端口 5173)
npm run dev
```

启动后访问 http://localhost:5173 即可打开平台。

### 3.4 环境变量配置

在项目根目录创建 `.env` 文件：

```env
# DeepSeek API 配置 (必填)
DEEPSEEK_API_KEY=your_api_key_here
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_LIVE=true

# RAG 配置 (可选)
RAG_BACKEND=hybrid          # "hybrid" | "keyword"
RAG_COLLECTION=springboot_course
RAG_CHUNK_SIZE=900
RAG_TOP_K=4
RAG_SNIPPET_SIZE=180
RAG_LESSON_BOOST=0.35
RAG_CONCEPT_BOOST=0.25
RAG_VECTOR_CANDIDATES=10

# 文件上传限制 (可选)
UPLOAD_MAX_BYTES=10485760   # 10MB
```

> 注意：如果没有 DeepSeek API Key，AI 对话和分析功能将无法使用，但其他功能（学习路线、自检答题等）仍可正常使用。

## 4. 学生端功能说明

### 4.1 登录

1. 打开平台首页，进入登录页面
2. 选择身份为"学生"
3. 选择所在班级（如"24软件本科班"）
4. 选择自己的姓名
5. 点击"进入学习"

登录后自动跳转到学习路线页面。登录状态会保存在浏览器中，关闭后再次打开无需重新登录。

### 4.2 学习路线

学习路线是学生端的首页，展示课程的学习地图。

**课程轨道切换：**
- 平台提供两条课程轨道，通过顶部标签切换
- **SpringBoot 12 讲**：从 Maven 基础到 SpringBoot 原理，共 12 课时
- **农宝系统后台管理项目**：基于真实项目的学习，5 个模块 12 个任务

**学习地图：**
- 采用不对称 Bento Grid 布局，视觉效果丰富
- 每个任务卡片显示：任务名称、类型、状态、进度
- 任务状态分为四种：
  - 已完成（绿色）：已通过 AI 验收
  - 进行中（蓝色）：当前可操作的任务
  - 未解锁（灰色）：前置任务未完成，暂不可操作
  - 需修改（橙色）：提交未通过，需要重新完成

**已发布题目：**
- 教师发布到该任务的题目会显示在任务卡片下方
- 点击题目可直接开始答题

**操作方式：**
- 点击"进行中"或"需修改"的任务卡片，进入任务工作台
- 点击已发布的题目，进入答题界面

### 4.3 任务工作台

任务工作台是学生完成学习任务的核心界面，采用左右双栏布局。

**左栏 — AI 对话区：**
- 输入框：输入问题或粘贴代码，按回车或点击发送
- AI 助教基于课程知识库回答，回答下方附带引用来源
- 支持 SSE 流式输出，AI 回答逐字显示

**右栏 — 代码/结果区：**
- 代码编辑区：粘贴或编写代码
- 验收结果区：显示 AI 验收的详细结果

**功能按钮：**

| 按钮 | 功能 | 说明 |
|------|------|------|
| 获取提示 | AI 给出一到两步提示 | 不会直接给完整答案，引导学生思考 |
| 学习反思 | AI 帮助复盘学习过程 | 基于当前任务生成学习总结 |
| 提交验收 | AI 基于 Rubric 评分 | 规则评分 60% + AI 评分 40% |

**提交验收流程：**
1. 在代码区填写完成的代码或答案
2. 点击"提交验收"
3. AI 根据任务的 Rubric 评分标准进行评判
4. 返回结构化结果：
   - 总分和通过状态
   - 亮点（做得好的部分）
   - 问题（需要改进的部分）
   - 下一步建议
   - 各项 Rubric 评分明细
5. 通过后自动解锁下一个任务

### 4.4 自检答题

自检答题用于检验学生对知识点的掌握程度。

**题型说明：**

| 题型 | 操作方式 | 评分方式 |
|------|---------|---------|
| 单选题 | 点击选择一个选项 | 自动批改 |
| 多选题 | 点击选择多个选项 | 自动批改 |
| 判断题 | 点击选择"正确"或"错误" | 自动批改 |
| 简答题 | 在文本框中输入答案 | AI 评分 |
| 代码填空 | 在代码编辑器中填写代码 | AI 评分 |

**答题流程：**
1. 在学习路线中点击已发布的题目，或进入任务的自检区域
2. 逐题作答
3. 点击"提交"查看结果
4. 系统自动批改客观题，AI 评分主观题
5. 成绩自动记录到学习档案

### 4.5 学习记录

学习记录页面展示学生的所有学习历史，按时间线排列。

**记录类型：**
- AI 对话记录：与 AI 助教的聊天历史
- 练习提交：代码练习的提交记录和 AI 反馈
- 自检成绩：自检答题的成绩记录
- 任务验收：AI 验收的结果记录
- 教师反馈：教师发送的文字反馈和 AI 分析结果

**教师反馈查看：**
- 教师反馈以特殊卡片样式展示
- 如附带 AI 分析结果，会显示优势、不足和改进计划

### 4.6 AI 配置

AI 配置页面用于管理 DeepSeek API 连接。

**配置项：**

| 配置项 | 说明 | 示例 |
|--------|------|------|
| API Key | DeepSeek API 密钥 | sk-xxxx |
| Base URL | API 地址 | https://api.deepseek.com |
| 模型 | 使用的模型名称 | deepseek-v4-flash |
| 超时时间 | 请求超时秒数 | 30 |
| 启用 AI | 是否启用 AI 功能 | 开/关 |

**操作步骤：**
1. 填写 API Key 和其他配置
2. 或点击预置模型列表一键填充
3. 点击"测试连接"验证配置是否正确
4. 点击"保存配置"

### 4.7 功能引导弹窗

首次进入平台时会自动弹出功能引导弹窗，展示学生端各功能模块的简要说明：
- 学习路线：选择课程轨道，查看任务状态
- AI 助教对话：向 AI 提问，获得引导式帮助
- 提交验收：Rubric 规则 + AI 双轨评分
- 自检答题：5 种题型，自动批改
- 学习记录：查看所有学习历史
- AI 配置：配置 DeepSeek API

勾选"下次不再显示"后，弹窗将不再自动弹出。

## 5. 教师端功能说明

### 5.1 登录

1. 打开平台首页，进入登录页面
2. 选择身份为"教师"
3. 选择所在班级
4. 选择自己的姓名
5. 点击"进入管理"

登录后自动跳转到仪表盘页面。

### 5.2 仪表盘

仪表盘是教师端的首页，展示班级的整体学习情况。

**顶部指标卡：**
- 学生总数：班级中的学生人数
- 今日活跃：今天有学习行为的学生数
- 整体完成率：所有任务的平均完成进度
- 需干预数：成绩低于阈值或长时间未活跃的学生数

**数据可视化图表：**
- 章节学习分布：各章节的学习进度柱状图
- 低分练习章节：薄弱知识点雷达图
- 提问热点关键词：学生提问的高频词汇词云
- 知识库来源类型占比：饼图展示知识库数据来源分布

**学生进度表：**
- 列出班级所有学生的学习进度
- 显示每个学生的完成率、平均分、最近活跃时间
- 点击学生可查看详细分析

**最近动态：**
- 展示最近的学习事件（提交、答题、聊天等）

**干预建议：**
- AI 基于数据生成的教学干预建议

### 5.3 知识库管理

知识库管理页面用于上传和管理课程资料，AI 助教的回答基于知识库内容。

**上传文件：**
1. 点击"上传文件"按钮
2. 选择本地文件
3. 系统自动解析文件内容并索引到 RAG 知识库

**支持的文件格式：**

| 类型 | 格式 |
|------|------|
| 文档 | PDF (.pdf), Word (.docx), Excel (.xlsx, .xls), CSV (.csv) |
| 代码 | .java, .py, .js, .ts, .vue, .jsx, .tsx, .xml, .sql, .html, .css |
| 配置/文本 | .md, .txt, .yml, .yaml, .json, .properties, .toml, .ini |

**文件管理：**
- 文件列表显示：文件名、文件类型、chunk 数量、上传时间
- 支持删除文件（同时删除 RAG 索引和任务关联）
- 支持将文件关联到具体学习任务

**关联任务：**
1. 在文件列表中选择一个文件
2. 选择要关联的任务
3. 点击"关联"
4. 关联后，AI 在处理该任务相关问题时会优先检索关联文件

### 5.4 题库管理

题库管理页面用于管理题目和发布题目到任务关卡。

**题库浏览：**
- 按课程轨道筛选题目
- 按章节筛选题目
- 显示题目列表：题型、题干、难度、标签

**新建题目：**
1. 点击"新建题目"
2. 选择课程和章节
3. 选择题型（单选/多选/判断/简答/代码填空）
4. 填写题干、选项（选择题）、正确答案、解析
5. 设置难度和标签
6. 保存

**编辑/删除题目：**
- 种子题库（系统预置）为只读，不可编辑或删除
- 教师自己创建的题目可以编辑和删除

**发布题目到任务关卡：**
1. 在题库中选择目标题目
2. 点击"发布"
3. 选择目标课程线和任务关卡
4. 确认发布
5. 发布后学生在对应任务的学习路线中可以看到并答题

**取消发布：**
1. 在已发布题目列表中找到目标
2. 点击"取消发布"
3. 学生端将不再显示该题目

### 5.5 学生管理

学生管理页面用于查看学生学习情况和进行 AI 分析。

**学生列表：**
- 按班级分组展示学生
- 显示每个学生的：姓名、学号、完成率、平均分

**单个学生 AI 分析：**
1. 在学生列表中点击"AI 分析"按钮
2. AI 自动分析该学生的学习数据（提交记录、交互历史、学习记录）
3. 返回结构化分析结果：
   - 优势：学生表现好的方面
   - 不足：需要改进的方面
   - 改进计划：具体的学习建议
   - 推荐任务：建议优先完成的任务
   - 总结：整体评价

**班级整体分析：**
1. 点击"班级分析"按钮
2. AI 分析整个班级的学习数据
3. 返回班级分析结果：
   - 共性弱项：班级普遍薄弱的知识点
   - 优秀学生：表现突出的学生
   - 困难学生：需要重点关注的学生
   - 教学建议：针对班级整体的教学改进建议

### 5.6 反馈中心

反馈中心用于教师向学生发送学习反馈。

**发送反馈：**
1. 选择目标学生
2. 输入反馈内容
3. (可选) 附带 AI 分析结果
4. 点击"发送"

**AI 分析转反馈：**
1. 在学生管理页面完成 AI 分析
2. 点击"发送给学生"
3. AI 分析结果自动填入反馈内容
4. 确认发送

**反馈历史：**
- 展示所有已发送的反馈记录
- 显示：发送时间、目标学生、反馈内容
- 学生可在"学习记录"中查看收到的反馈

### 5.7 AI 配置

教师端的 AI 配置页面功能与学生端相同，但采用马卡龙 HF 视觉风格。

**额外功能 — 系统状态：**
- RAG 知识库状态：后端类型、chunk 数量
- 数据库连接状态
- DeepSeek 模型配置信息
- 文件上传目录状态

## 7. 技术架构简图

```
+------------------+     HTTP/SSE     +------------------+
|                  | <--------------> |                  |
|   前端 (Vue 3)   |                  |  后端 (FastAPI)   |
|                  |                  |                  |
|  - Vue Router    |                  |  - API 路由层     |
|  - Pinia 状态    |                  |  - 服务层         |
|  - Element Plus  |                  |  - LangGraph     |
|  - ECharts       |                  |  - RAG 检索       |
|  - Axios         |                  |  - Rubric 引擎    |
+------------------+                  +--------+---------+
                                               |
                                               |
                                     +---------+---------+
                                     |                   |
                                     |  SQLite 数据库     |
                                     |  (14 张表)         |
                                     |                   |
                                     |  ChromaDB (可选)   |
                                     |  (向量索引)        |
                                     |                   |
                                     |  文件系统          |
                                     |  (课程资料/上传)   |
                                     +-------------------+
                                               |
                                               |
                                     +---------+---------+
                                     |                   |
                                     |  DeepSeek API     |
                                     |  (LLM 服务)       |
                                     +-------------------+
```

**数据流向：**
1. 学生在前端操作（提问、提交代码、答题）
2. 前端通过 HTTP/SSE 发送请求到后端
3. 后端从 RAG 知识库检索相关内容
4. 后端调用 DeepSeek API 生成回答或评分
5. 结果返回前端展示，同时记录到 SQLite 数据库
6. 教师端读取数据库数据进行分析和管理

## 8. 常见问题 FAQ

### Q1: 启动后 AI 功能不可用怎么办？

A: 请检查以下几点：
1. 确认 `.env` 文件中已配置正确的 `DEEPSEEK_API_KEY`
2. 确认 `DEEPSEEK_LIVE=true`
3. 在 AI 配置页面点击"测试连接"验证 API 是否可用
4. 检查网络连接是否正常

### Q2: 上传文件后 AI 没有使用新内容回答怎么办？

A: 上传文件后需要等待系统完成索引。如果仍未生效：
1. 检查文件格式是否受支持
2. 尝试在知识库管理页面点击"重新索引"
3. 确认文件内容不为空

### Q3: 任务显示"未解锁"无法点击怎么办？

A: 任务有前置依赖，需要完成前置任务并通过 AI 验收后才能解锁。请按照学习路线的顺序依次完成任务。

### Q4: 提交验收总是不通过怎么办？

A: 验收采用 Rubric 规则评分，建议：
1. 仔细阅读任务说明和评分标准
2. 使用"获取提示"功能获取引导
3. 检查是否有必选标准（required）未满足
4. 查看验收结果中的"问题"和"下一步建议"

### Q5: 学生端和教师端的 AI 配置是独立的吗？

A: 不是。学生端和教师端共用同一个 `.env` 配置文件，任何一端修改都会影响另一端。

### Q6: 如何添加新的课程轨道？

A: 目前课程轨道在 `backend/app/data/courses.py` 和 `backend/app/data/course_runtime_seed.py` 中定义。添加新轨道需要修改这些数据文件并重启后端服务。

### Q7: 数据库文件在哪里？

A: SQLite 数据库文件位于 `backend/.data/learning.sqlite`。首次启动时自动创建，包含所有学习记录、用户信息、任务进度等数据。

### Q8: 如何重置所有数据？

A: 删除 `backend/.data/learning.sqlite` 文件后重启后端服务，系统会自动重新初始化数据库和种子数据。

### Q9: 前端页面样式异常怎么办？

A: 尝试以下步骤：
1. 清除浏览器缓存
2. 确认使用的是现代浏览器（Chrome/Edge/Firefox 最新版）
3. 检查控制台是否有错误信息
4. 重新执行 `npm install` 和 `npm run dev`

### Q10: ChromaDB 向量检索如何启用？

A: 执行以下步骤：
1. 安装可选依赖：`pip install -r backend/requirements-optional-rag.txt`
2. 在 `.env` 中设置 `RAG_BACKEND=hybrid`
3. 重启后端服务
4. 系统会自动使用关键词 + 向量混合检索
